## The **counter-dapp** program

[![Build Status](https://github.com/gear-tech/counter-dapp/workflows/CI/badge.svg)](https://github.com/gear-tech/counter-dapp/actions)

Program **counter-dapp** for [⚙️ Gear Protocol](https://github.com/gear-tech/gear) written in [⛵ Sails](https://github.com/gear-tech/sails) framework.

The program workspace includes the following packages:
- `counter-dapp` is the package allowing to build WASM binary for the program and IDL file for it.
  The package also includes integration tests for the program in the `tests` sub-folder
- `counter-dapp-app` is the package containing business logic for the program represented by the `CounterDapp` structure.
- `counter-dapp-client` is the package containing the client for the program allowing to interact with it from another program, tests, or off-chain client.

### 🏗️ Building

```bash
cargo build --release
```

### ✅ Testing

```bash
cargo test --release
```

### Counter dApp

The program exposes a `counter` service:

- `increment() -> i32`
- `decrement() -> i32`
- `value() -> i32`
- `reset() -> Result<(), CounterError>`

`reset()` is owner-gated. The owner is the account that calls the `create()` constructor.

Build artifacts for IDEA portal deployment are generated by:

```bash
cargo build --release
```

Upload this Wasm in IDEA portal:

```text
target/wasm32-gear/release/counter_dapp.opt.wasm
```

Use this IDL when needed:

```text
client/counter_dapp_client.idl
```

Frontend:

```bash
cd frontend
npm install
npm run dev
```

Set `frontend/.env` before opening the app:

```text
VITE_GEAR_ENDPOINT=wss://testnet.vara.network
VITE_PROGRAM_ID=<deployed program id>
```

> For off-chain integration tests against a running node, add the `gclient` feature:
>
> ```bash
> cargo add sails-rs --dev --features gclient
> ```
